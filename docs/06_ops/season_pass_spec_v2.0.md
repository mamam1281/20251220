차# 레벨 2차 스펙 v2.0 (2025-12-15, KST)

## 1) 목적
- 12/15(월) 시작 레벨 2차 운영안 확정: 레벨 커브·보상 표준화, FE/BE/QA 공통 참조.
- 대상: 백엔드·프론트엔드·QA·운영.

## 2) 일정/타임존
- 시즌 기간: **2025-12-15 00:00 KST 시작** / 종료일 운영 확정 후 반영 (기본 14일 권장).
- 모든 날짜/리셋/표시는 Asia/Seoul 기준.

- 스탬프당 기본 XP: 20 (변경 시 BE config 필요).

## 4) 보상 정책
| 레벨 | 보상 | 수량/내용 | 지급 방식 제안 |
|---|---|---|---|
| 1 | 룰렛 티켓 | 1장 | 자동 |
| 2 | 주사위 티켓 | 1장 | 자동 |
| 3 | 룰렛 티켓 1 + 주사위 티켓 1 | 1세트 | 자동 |
| 4 | 복권 티켓 | 1장 | 자동 |
| 5 | CC 코인 | 1개 | 자동 |
| 6 | 주사위 티켓 2 + 복권 티켓 1 | 1세트 | 자동 |
| 7 | CC 코인 | 2개 | 자동 |
| 8 | 쿠팡상품권 1만원 | 1매 | **수동 클레임(운영 검증 필요)** |
| 9 | CC 포인트 2만 | 1건 | **수동 클레임(포인트 지급 확인)** |
| 10| CC 포인트 5만 | 1건 | **수동 클레임(포인트 지급 확인)** |

- 쿠폰/포인트 류는 운영 검증 필요하므로 auto_claim=0 권장. 티켓/코인은 auto_claim=1 권장.
- 실제 지급 타입 매핑 예시: 룰렛 티켓=ROULETTE_TICKET, 주사위 티켓=DICE_TICKET, 복권 티켓=LOTTERY_TICKET, CC 코인=CC_COIN, CC 포인트=CC_POINT.

## 5) 기본 규칙
- 하루 도장 1회 한정(KST), 중복 시 400.
- xp_to_add = base_xp_per_stamp(20) + xp_bonus(호출자 입력). 음수 불가.
- 다단계 승급 시 새로 달성한 모든 레벨의 auto_claim=1 보상 즉시 지급, auto_claim=0은 수동.
- 로그 유니크: stamp_log (user_id, season_id, date), reward_log (user_id, season_id, level).
- 활성 시즌 다중 발생 시 409 NO_ACTIVE_SEASON_CONFLICT.

## 6) 운영/QA 체크리스트
- [ ] season_pass_config: start/end, base_xp_per_stamp=20, max_level=10.
- [ ] season_pass_level: 위 XP/보상 표대로 seeding, auto_claim 플래그 반영.
- [ ] FE 노출: 레벨 커브/보상표/자동·수동 클레임 문구 확인.
- [ ] BE: /api/season-pass/status, /stamp, /claim 정상 응답 및 timezone KST 확인.
- [ ] 정기 점검: 종료일 하루 전 보상 미수령 사용자 알림 여부 (선택).

## 7) 변동/확정 포인트
- 종료일: 확정 후 config 반영 필요.
- 상품권 지급 방식: 운영/재무 프로세스 확정 후 auto_claim 여부 재검토.
- 포인트 지급 채널(CC 포인트) 확인: 지급 실패 시 리트라이/보류 정책 필요.
